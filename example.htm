jQuery Plugin : validationEngine [VE2]

Description

One of the greatest challenges for anyone who builds, owns or manages a website, especially one which allows users to interact with it using forms, is ensuring that the user enters the right data in the right fields.

Users, at the same time, suffer when the validation of their submissions are all done server-side (where there is a longer round-trip delay), where the error messages are not displayed in context, or where the errors are obscure, confusing, or otherwise frustrating.

This plugin, and those which are it's ancestors - validationEngine by JOE and validationEngine2 by jk, allow for the client-side validation of form data with error messages displayed alongside the fields in question. It allows for basic validation checks (non-empty fields), regular expression matches, AJAX validations, custom functions, etc. And, whatever it doesn't already have can always be added, either by contacting me, or by forking the plugin at GitHub.

Usage

Required Field

An error message will be displayed if the user moves from the field without entering any value.

Sandbox

HTML<form id="example1"> <label>This is a Required Field: <input name="field1" class="validate[required]" /></label><br/> <label> This is a Required Dropdown: <select name="field2" class="validate[required]"> <option value="" selected="selected">Please select one... (has a value of "")</option> <option>I am Juan, you want to select me?</option> </select> </label> </form> Javascript$(document).ready(function(){ $('#example1').validationEngine({ promptPosition : 'bottomLeft' }); });
